Part 2 · Chromatic stack — two layers, third color as bg

Two colored copies of the lockup — a front color and a single offset shadow — set against the third palette color as the background. Each combination uses all three palette colors but distributes them across two roles (typography) plus one role (field). Halved offset (4px default, 7px hero) for a tighter chromatic effect than the previous version. Order code is front-shadow; the bg is whichever of the three colors isn't named.

Notes:

• Chromatic stack uses pure CSS (text-shadow) — no extra font requests, works with any display face.

• Each (lockup, bg) pairing uses all three palette colors by construction — the two lockup colors are different, and the bg is the third. No collision possible. The "for" connector defaults to cream/navy depending on bg legibility.

• The lockup on cream bg (the default site background) is handled by the small inline wordmark in the navigation — we don't render the chromatic stack on cream because it requires a third color in play, and chromatic-on-cream would either need a 4th color or a different technique. The chromatic stack lives on hero blocks, in-focus sections, and footers — places where one of the three palette colors is already the field.
